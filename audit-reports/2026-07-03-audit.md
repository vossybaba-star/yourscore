## Medium

### M-invite (carryover, unchanged) — `/api/room/invite` fires push to ANY user, no friend check, no rate limit
**Where:** [src/app/api/room/invite/route.ts](src/app/api/room/invite/route.ts) — re-read in full today. Authed and correctly checks `room.created_by === user.id`, but still takes an attacker-chosen `invitedUserId`, does **no** friend/relationship check, and has **no `rateLimitDistributed`**. [src/app/api/room/create/route.ts](src/app/api/room/create/route.ts) is likewise unrate-limited. Route untouched since `a6c8447`.
**Abuse path:** loop `room/create` → `room/invite {roomId, invitedUserId: <stranger>}`; a fresh room per call defeats the `room-invite:${room.id}:${invitedUserId}` dedup → unbounded pushes + "wants to play you live" inbox spam to any opted-in user (target ids enumerable via anon leaderboard / `h2h_challenges`). Still the most actionable push finding.
**Fix:** verify a mutual/accepted friend before targeting `invitedUserId`; add `rateLimitDistributed` to both `room/invite` and `room/create`; optionally a per-target-per-day notify cap independent of the dedupe key.
