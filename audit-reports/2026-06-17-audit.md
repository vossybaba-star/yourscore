# YourScore Security & Code Audit — 2026-06-17

**Branch/HEAD:** `main` @ `c67d2a6` (fix(ios): password-reset deep link + Universal Links groundwork)
**Scope:** ~40 commits since the last audit — almost all the new "World Cup Daily" ranked competition (server-authoritative draft, knockout deciders, season board, player-history drill-down) plus iOS deep-link / Universal Links work.

## Executive summary
The new WC ranked **server architecture is genuinely well-built**: every mutating route is authenticated, scoped by `user_id` (no IDOR), rate-limited, server-authoritative (opponents, goals, draft slates, quiz grading all resolved server-side), and the draft slate seed is HMAC-peppered so it can't be precomputed. The prior open-redirect (M1) is now correctly fixed, and `/api/club-preview` is admin-locked.

**However, one Critical defect undermines all of that for the £100 prize board:** the entire World Cup quiz answer bank (63 questions, with correct-answer indices) is **bundled into the client JavaScript**. Anyone with DevTools can map any "server-graded" question to its answer, defeating the ranked draft, the Mastermind score, and the knockout deciders — a direct fraud path on the cash board. Separately, an **untracked `.env.local.bak.*` secrets backup is not covered by `.gitignore`** (one `git add -A` from being committed), and the prior **H1 anon-key quiz-answer exposure remains open**.

## Critical

### C1 — WC quiz answer bank is shipped to the browser (defeats ranked prize-board integrity)
**Files:** `src/lib/draft/wc-quiz.ts:11` (`import bundle from "@/data/draft/wc-quiz.json"`, no `server-only`); imported by client components `src/app/38-0/wc/run/[id]/page.tsx:21` and `src/app/38-0/wc/page.tsx`. Bank: `src/data/draft/wc-quiz.json` — 63 questions, each with `"answer": <index>`.

**What's wrong:** `wc-quiz.ts` has no `import "server-only"` and is imported by two `"use client"` pages (for `drawQuestion`, the practice/upgrade re-spin). Webpack therefore bundles the full JSON — *including every `answer` index* — into the client chunk. The server carefully strips `correctIndex` before sending decider/draft questions, but the answers are already sitting in the page's JS.

**Why it matters:** This is the exact "quiz answers reachable by the client" fraud class as H1, but for the **cash WC2026 prize board**, and it needs no API knowledge — just view-source. A cheater can:
- **Ranked draft:** for each served question, match `prompt` → bundled question → correct option text → submit the matching shuffled index. Every answer correct ⇒ band always maxes ⇒ strongest possible XI ⇒ inflated wins on the season board. `verifyRankedDraft` only checks picks were *server-offered options for the earned band* — it cannot tell the answers were cheated.
- **Knockout / play-off deciders:** the server sends `{prompt, options}` with no index; the client looks the prompt up in the bundle and always picks correctly ⇒ never loses a tie.
- **Mastermind score (`quiz_correct`):** trivially maxed.

**Compounding leak (see M1):** `rowToRun` includes `seed`, and `/play`, `/decide`, `/[id]` return that `run` object — so the run seed reaches the client. With the bundled `deciderQuestion(seed)` + bundled POOL, deciders can be precomputed even without prompt-matching.

**Suggested fix:** Get the answer bank out of the client bundle. Split `wc-quiz.ts`: keep a tiny client module with only the types + a client-safe `drawQuestion` that operates on questions the *server* hands down (answers stripped), and move `POOL`/`deciderQuestion`/`dailyQuestions`/`rankedQuizScore` into a `server-only` module. For the client-side practice/upgrade re-spin, either (a) grade it server-side too, or (b) accept that practice is low-stakes but still don't ship the *ranked* bank — practice questions can be served answer-stripped from an API. Net rule: **no file that imports `wc-quiz.json` may be reachable from a `"use client"` component.** Add a quick CI grep to enforce it.

## High

### H-prior (CARRYOVER, still open) — anon key can read quiz answer columns
**Files:** `supabase/schema.sql:422` (`questions` `for select to public using (true)`), `:426` (`quiz_packs`), `:385` (`challenge_questions`).
**What's wrong / why it matters:** Unchanged since Jun 12. The public anon key can `SELECT` answer columns on the daily-quiz tables → scripted cheating on the prize quiz board. This is the structural twin of C1 (different game surface).
**Suggested fix:** As previously scoped — strip answer columns from public reads / serve gameplay questions via a server route that omits the answer, or split answers into an anon-revoked table. The `audit-wc2026-integrity.mjs` payout gate remains the compensating control until then.

### H1 — `.env.local.bak.1781514469` is not gitignored (secret backup at risk of commit)
**File:** repo root `.env.local.bak.1781514469` (untracked). `.gitignore:29` is `.env*.local`, which matches `.env.local` but **not** the `.bak.<n>` suffix (`git check-ignore` confirms: NOT ignored).
**What's wrong / why it matters:** A full backup of `.env.local` (Supabase service role, Sentry token, ad pixels, Resend key, etc.) sits in the working tree and would be staged by any `git add -A` / `git add .`, then potentially pushed to GitHub — a credential leak.
**Suggested fix:** Delete the backup now (`rm .env.local.bak.1781514469`), and harden `.gitignore` with `.env*` and `*.bak` so future backups can't be committed. If it was ever staged/pushed, rotate the affected keys.

## Medium

### M1 — Run `seed` leaked to the client
**Files:** `src/lib/draft/wc-server.ts:64` (`rowToRun` maps `seed`); returned via `src/app/api/draft/wc/play/route.ts:41,54,58`, `decide/route.ts:47,51`, `[id]/route.ts:32`.
**What's wrong:** The client-facing `run` payload carries the full run seed. The client `Run` type doesn't even declare it, so it's unused-but-exposed data. Combined with the bundled `deciderQuestion`/opponent logic, it lets a client precompute deciders and opponent XIs.
**Why it matters:** On its own, low impact (opponents/goals are still server-resolved); but it's an unnecessary secret over the wire that meaningfully compounds C1.
**Suggested fix:** Add a `runToClient(run)` projection that omits `seed` (and any other server-only fields) and use it in all WC route responses.

### M2 — Migration numbering collision + uncommitted RLS migration (drift risk)
**Files:** `supabase/migrations/43_draft_wc_ranked_edition.sql` (committed) **and** `supabase/migrations/43_rls_security_advisor_fixes.sql` (untracked, applied to prod Jun 17 per prior audit). Two migrations share number `43`.
**What's wrong / why it matters:** Duplicate ordinals break the assumption that migration order is total/unambiguous; a fresh `supabase db reset` or a new environment could apply them in the wrong order or skip one. The RLS advisor-fix migration (enables RLS + revokes anon/auth grants on `admin_chat_messages`, `spend_logs`, `health_logs`) is **applied to prod but never committed** — if someone rebuilds from the repo, those security fixes silently vanish.
**Suggested fix:** Renumber the RLS file to `44_rls_security_advisor_fixes.sql` and **commit it** so prod state is reproducible from the repo.

### M3 — Anti-preview lock only commits after the 6th pick
**File:** `src/app/api/draft/wc/draft/route.ts:21` (`LOCK_AFTER = 6`), `:60-74`.
**What's wrong:** The `slate` action returns `correctIndex` for each answered question (to render feedback), but the day's ranked attempt isn't locked until pick index ≥ 6. A player can step through picks 0–5, read those `correctIndex` values, bail before the lock, and restart the ranked draft knowing the first 6 answers.
**Why it matters:** Partial answer preview on the ranked board. (Largely subsumed by C1 — once C1 is fixed this is the remaining preview gap.)
**Suggested fix:** Commit the lock on the *first* slate call of a ranked attempt (or don't return `correctIndex` for ranked until submit). Track ranked vs practice explicitly in the slate request so practice keeps its instant feedback.
