## High

### H1 ‚Äî `profiles.timezone` / `country` location PII is anon-readable and now LIVE (escalated from yesterday's M-new)
**Where:** [supabase/migrations/57_profile_timezone.sql](supabase/migrations/57_profile_timezone.sql) adds `timezone` + `country` to `profiles` with **no column-level REVOKE**, so they inherit the permissive public `profiles` SELECT policy.

**Confirmed live this audit (anon key only):**
- `140 / 5,952` profiles now have populated `timezone` (and `country`) ‚Äî yesterday they were all null, so this is no longer hypothetical.
- Real values returned: `Europe/London`/`GB`, `Asia/Bangkok`/`TH`, etc.
- **Joinable to public identity in one query:** `select=username,display_name,timezone,country` returns e.g. `{"username":"babavossy", "timezone":"Asia/Bangkok", "country":"TH"}`. So this is named, per-user coarse location, enumerable in bulk with the public key.

**Why it matters:** location inference keyed to a named account, harvestable at scale ‚Äî a clear privacy regression versus what `profiles` exposed before, and it grows every time the app runs `<TimezoneSync/>`. These columns are only ever read server-side (the hourly cron computes local send windows); anon / other-user reads are never needed.

**Fix:** column-level `REVOKE timezone, country, active_hour_utc ON public.profiles FROM anon, authenticated;` (the cron uses the service role and is unaffected), or move the notification-targeting columns to a service-role-only sidecar table. Pair with a one-line migration so the grant state is captured in the repo, not just applied out-of-band.

### H-prior ‚Äî anon key can read quiz `answer` columns (fraud path on the ¬£100 board)
**Where:** `questions` (and `quiz_packs`) public SELECT policy `USING(true)`.
**Re-confirmed live:** `GET /rest/v1/questions?select=id,answer` with the anon key returns plaintext answers (`"A"`, `"B"`, ‚Ä¶) for every question. A scripted client can pre-fetch answers for any quiz-gated content, defeating ranked-band / Mastermind integrity. Open since Jun 12.
**Fix:** strip `answer` (and decider columns) from anon-readable grants ‚Äî column-level `REVOKE`, or serve questions through a server route that omits the answer and grades server-side only.
