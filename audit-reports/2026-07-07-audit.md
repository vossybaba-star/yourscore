## High

### H-attribution (NEW) — anon key can read every user's acquisition source, campaign IDs and referrer
**Where:** [supabase/migrations/74_profile_acquisition_source.sql](supabase/migrations/74_profile_acquisition_source.sql) added `source, utm_source, utm_medium, utm_campaign, referrer` to `profiles`. The `profiles` SELECT policy is `"Public profiles readable" USING(true)` for `PUBLIC`, and column-level SELECT on all five new columns is granted to **`anon`** and **`authenticated`** (confirmed via `information_schema.column_privileges`).
**Proof (public anon key, read live):**
```
GET /rest/v1/profiles?select=id,source,utm_source,utm_campaign,referrer&source=not.is.null
→ [{"source":"meta","utm_source":"ig","utm_campaign":"120250560138900130",...},
   {"source":"x","referrer":"https://t.co/",...}, …]   Content-Range: 0-69/70
```
**Why it matters:** the anon key ships in the client bundle, so anyone can enumerate **which platform/campaign acquired each user** and the **exact Meta campaign IDs** you're spending on — competitive-intelligence gold for a rival, and `referrer` can carry mildly identifying inbound URLs. It's a fresh regression: the attribution feature widened the anon read surface the same day it landed. Same root cause as the standing H-pii item (`USING(true)` profile reads), so it's the same family, but net-new data.
**Fix:** `REVOKE SELECT (source, utm_source, utm_medium, utm_campaign, referrer) ON public.profiles FROM anon, authenticated;` — these columns are only ever written (service role) and read server-side for analytics; no client path needs them. Longer-term, move `profiles` off blanket `USING(true)` to an explicit public-column projection (view or column-scoped grants) so the next added column isn't auto-exposed.

### H-answers (carryover, STILL OPEN) — anon key reads quiz answer columns
**Where:** `questions` + `quiz_packs` SELECT `USING(true)`. **Verified live today:** `GET /rest/v1/questions?select=id,question,answer` returns `"answer":"C"`; `quiz_packs.questions` returns the embedded `"answer"` per question. The original £100-board fraud path. Compensating control = the payout-integrity script (`scripts/audit-wc2026-integrity.mjs`); structural fix deferred deliberately (touches all three game-loop god files). Unchanged.

### H-pii (carryover, STILL OPEN, now broader) — anon-readable profile fields
`profiles.country` / `timezone` still anon-readable; H-attribution above adds five more columns under the same policy. Fix them together with a column-scoped grant model.
