## Critical (carryover — STILL OPEN, top integrity priority)

### C1 — WC quiz answer bank still shipped to the browser
Re-confirmed: `src/lib/draft/wc-quiz.ts` still **lacks `import "server-only"`** and is imported by two `"use client"` pages (`src/app/38-0/wc/page.tsx`, `src/app/38-0/wc/run/[id]/page.tsx`). The 63-question bank with `answer` indices is bundled into client JS, defeating ranked-board integrity on the £100 prize. Unchanged since Jun 17.
**Fix:** split `POOL`/`deciderQuestion`/`dailyQuestions` into a `server-only` module; add a CI grep forbidding any client import of `wc-quiz.json`.

## High (carryover — STILL OPEN)

### H-prior — anon key can read quiz-answer columns
Re-confirmed in `supabase/schema.sql`: `challenge_questions` (`:385`), `questions` (`:422`), `quiz_packs` (`:426`) are all `for select to public using (true)` → the anon key can `SELECT` answer columns, enabling scripted cheating on the prize board. `scripts/audit-wc2026-integrity.mjs` remains the compensating control.
**Fix:** serve gameplay questions via a server route that omits answers, or split answers into an anon-revoked table.

### H1 — daily quiz email still on the over-quota *transactional* Resend pool
`scripts/send-wc-quiz-daily.mjs` still fans the daily all-users send out via `RESEND_API_KEY` (transactional pool, last recorded over quota at 57,676/50,000), not Broadcasts. ~3,275 sends/day share the cap with password resets / magic links. Carried — no code change this cycle.
**Fix:** migrate the daily send to Resend Broadcasts (as the mastermind campaigns already use).

## Medium (carryover — STILL OPEN)

### M1 — daily push log-before-send can silently drop a whole cohort
`src/lib/notify.ts:55-79` inserts `notification_log` dedup rows **before** awaiting `send-push`. Because the cron targets each user in exactly one UTC hour and dedups per-day (`wc-mastermind:<date>`), any transient `send-push` failure during that hour permanently loses that cohort's daily ping with no retry. Unchanged since first flagged Jun 24.
**Fix:** insert the log **after** a confirmed 2xx (accept rare at-least-once), or delete the just-inserted rows on non-2xx; at minimum surface a `delivered` count instead of swallowing failures.

### M2 — run `seed` leak / `LOCK_AFTER=6` preview window
Carried unchanged from Jun 17; not re-walked this cycle. `rowToRun` still returns `seed` (deciders precomputable) and the anti-preview lock only engages after pick 6.

## Low (carryover — STILL OPEN, hygiene)
